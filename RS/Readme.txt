===============================================================================
| This directory is the root from the directory structure of the RedScarf
| front-end.
===============================================================================

===============================================================================
| In this directory is only the Qt project file of the front-end (RS.pro) and
| the resources subdirs of the tool. That are:
| -----------------------------------------------------------------------------
| * bin          : Place where the front-end executable is generated.
|                  NOTE: This directory has the back-end of the tool and the
|                  environment configuration files in their subdirs.
| -----------------------------------------------------------------------------
| * build        : Place to files resulting from the compilation process, as:
|                  objects (.o), source-code of the forms (ui_*.h files generated
|                  from the .ui files), files from Qt meta compilation (moc_*.cpp)
|                  and resources files (generated by .qrc files).
| -----------------------------------------------------------------------------
| * dist         : Place to keep records of changes (change logs), configuration
|                  files and scripts to generate the "installers" for target
|                  platforms.
| -----------------------------------------------------------------------------
| * doc          : Place to keep the project documentation.
| -----------------------------------------------------------------------------
| * forms        : Place of graphical forms of the front-end.
| -----------------------------------------------------------------------------
| * include      : Place of source-code headers (.h). This directory is divided
|                  in 3 subdirs: Model, View and Control, because the front-end
|                  was divided in layers of the Object-Oriented MVC model.
| -----------------------------------------------------------------------------
| * resources    : Place of front-end resources (resource files .qrc, icons,
|                  sounds, ...).
| -----------------------------------------------------------------------------
| * src          : Place of front-end source-codes (.cpp). This directory is
|                  divided in 3 subdirs: Model, View and Control, because the
|                  front-end was divided in layers of the Object-Oriented MVC
|                  model.
| -----------------------------------------------------------------------------
| * translations : Place of the files to translate the front-end.
===============================================================================

===============================================================================
| NOTE: Don't add anything to this directory, always keep organized the directory
| structure and each feature created, put it in the suitable location.
===============================================================================
| During the compilation, some files are created in this directory, it isn't
| necessary remove them, but be care to don't add this files in the code
| versioning.
===============================================================================
| To get more information about each directory look at the directory Readme file.
| NOTE: The "src", "include", "forms" and "resources" directories don't have
| Readme file and the files are acessible by Qt Creator IDE when open the root
| project of the framework - RedScarf.pro
===============================================================================
